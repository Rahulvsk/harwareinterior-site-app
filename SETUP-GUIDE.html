Part 1 — Host the website

Step 1. Create accounts (free unless noted)

GitHub, MongoDB Atlas, Railway, Vercel.

Step 2. Push code to GitHub PRIVATE REPO

Your .env files hold real secrets — keep the repo private.

cd "Main_Folder"
git init
git add .
git status        # confirm NO .env files appear (only .env.example)
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/<you>/<repo>.git
git push -u origin main

Step 3. MongoDB Atlas

  1. Create or open a cluster.
  2. Database Access → create a user → set a NEW password (the old one was exposed; rotate it).
  3. Network Access → add IP 0.0.0.0/0 so Railway can connect.
  4. Connect → Drivers → copy the connection string = your MONGO_URI.

Step 4. Deploy backend on Railway DO FIRST

  1. railway.app → New Project → Deploy from GitHub repo.
  2. Service → Settings → Root Directory = backend.
  3. Railway runs npm install then npm start automatically.
  4. Variables → add backend env vars (don't set PORT): MONGO_URI, JWT_SECRET, JWT_ACCESS_LIFETIME, JWT_REFRESH_SECRET, JWT_REFRESH_LIFETIME, JWT_SECRET_FOR_VERIFY, ENCRYPT_PASSWORD, STRIPE_KEY, MAX_AMOUNT, MIN_AMOUNT, AMOUNT_STEP, CURRENCY, STRIPE_PAYMENT_DESCRIPTION, SERVICE, EMAIL_USER, EMAIL_PASS, HOST, EMAIL_PORT, STORE_URL, ADMIN_URL.
  5. Settings → Networking → Generate Domain. e.g. https://your-backend.up.railway.app.
  6. API base URL = that domain + /v1 — reused everywhere.

Step 5. Deploy admin on Vercel

  1. vercel.com → Add New → Project → import repo.
  2. Root Directory = dashtar; framework Vite. A vercel.json for SPA routing is already included.
  3. Env vars (VITE_*): VITE_APP_API_BASE_URL = https://your-backend.up.railway.app/v1, VITE_APP_API_SOCKET_URL = https://your-backend.up.railway.app.
  4. Deploy → set VITE_APP_ADMIN_DOMAIN = https://<admin-url>/login → redeploy.

Step 6. Deploy store on Vercel

  1. Add New → Project → same repo → Root Directory = store → framework Next.js.
  2. Env vars (NEXT_PUBLIC_* + NEXTAUTH_*) with API base pointed at Railway.
  3. Deploy → set NEXT_PUBLIC_STORE_DOMAIN + NEXTAUTH_URL to the store URL → redeploy.

Step 7. Wire all three together

  • Railway backend: ADMIN_URL = admin URL, STORE_URL = store URL.
  • Vercel admin: VITE_APP_STORE_DOMAIN = store URL, VITE_APP_ADMIN_DOMAIN = admin URL + /login.
  • Vercel store: NEXT_PUBLIC_STORE_DOMAIN + NEXTAUTH_URL = store URL.

Step 8. First login

Open the admin site → Sign Up to create your first admin → log in. (Or seed locally: in backend/ with production MONGO_URI, run npm run data:import.) ✅ Website live.

Part 2 — The mobile apps

Step 9. Point apps at your live backend

Edit both mobile/customer-app/app.json and mobile/admin-app/app.json:

"extra": { "apiBaseUrl": "https://your-backend.up.railway.app/v1" }

Commit and push. (For local phone testing use your computer's LAN IP, e.g. http://192.168.1.20:5055/v1 — not localhost.)

Step 10. Get the APK from GitHub Actions NO PAID ACCOUNT

Two workflows already live in .github/workflows/ and build an installable Android APK in the cloud.

  1. Push the repo to GitHub (Step 2) — includes the workflow files.
  2. On GitHub open the Actions tab.
  3. Pick “Build Customer APK” (or “Build Admin APK”) → Run workflow → branch main → Run.
  4. Wait ~10–20 min for the green check.
  5. Open the finished run → Artifacts → download customer-app-apk / admin-app-apk.
  6. Copy the .apk to an Android phone and open it to install (allow “Install unknown apps”).
These APKs are debug-signed — great for testing & sharing directly, but not accepted by the Google Play Store. For the store, use a release build (Step 12).

Step 11. Test instantly on your phone (optional)

cd mobile/customer-app   # or mobile/admin-app
npm install
npx expo start

Install Expo Go, scan the QR code, app runs live with hot reload.

Step 12. Publish to App Store & Play Store

npm install -g eas-cli
eas login
cd mobile/customer-app          # repeat for admin-app
eas build:configure
eas build --platform android    # .aab for Play Store
eas build --platform ios        # .ipa for App Store
eas submit --platform android
eas submit --platform ios

Paid accounts (only you can create): Apple Developer $99/year (App Store), Google Play $25 one-time (Play Store), Expo account free (EAS). Before submitting: add a 1024×1024 icon + splash, set app name, bump version.

Troubleshooting

  • App shows no products: wrong apiBaseUrl or backend unreachable. Confirm the Railway URL ends in /v1 and opens in a browser.
  • Admin login fails on mobile: ensure an admin account exists and the API URL is the live backend.
  • GitHub Action fails at “Build APK”: read the failed step's logs — usually an app.json typo or a dependency mismatch (run npx expo install locally to sync).
  • Env change didn't apply: Vercel/Railway need a redeploy after editing variables.
  • Secrets: never commit .env. Rotate the MongoDB password, Gmail app password, and Google OAuth secret (all were exposed earlier).

Quick reference — deploy order

1. GitHub (private) → 2. Atlas (rotate password) → 3. Railway backend → 4. Vercel admin → 5. Vercel store → 6. wire URLs → 7. set app apiBaseUrl → 8. run GitHub Action → download APK.

Host the backend first — the website frontends and both mobile apps all depend on its URL.
